블록/인라인 요소

html의 모든 요소는 기본적으로 display 속성에 의해 웹 브라우저로 어떻게 보여질지 정해져 있음

display 속성 값이 block인 경우 요소의 너비는 언제나 100%이고, 새로운 라인에 배치되며, width, height속성으로 크기를 임의로 바꿀수 있고, margin, padding으로 여백도 변경할 수 있음 (h1~h6,p,div,hr,ul,ol,form)

시간은 금이라구, 친구!

에바에 타라, 신지!

display 속성 값이 inline인 경우 요소의 너비는 내용에 따라 달라지고, width, height속성으로 크기를 임의로 바꿀수 없고, margin, padding으로 여백도 변경할 수 없음 (span,문자관련태그,img,a)

시간은 금이라구, 친구! 에바에 타라, 신지!

블록 vs 인라인

블록 요소는 다른 요소들을 하나로 묶어 화면 배치를 구성하는데 주로 사용

또한, 관련 요소들을 하나로 묶어 스타일을 일괄적으로 적용할때도 사용

반면, 인라인 요소는 텍스트 요소들을 하나로 묶어 스타일을 일괄적으로 적용할때 사용

블록요소와 인라인요소를 함께 사용할때는 반드시 블록요소를 먼저 사용하고, 그 안에 인라인요소를 포함시켜야 함

html 온라인 문법 검사기로 html 소스를 적절하게 작성했는지 알아볼수도 있음 : validator.w3.org

시간은 금이라구, 친구!!

레이아웃

레이아웃은 웹 페이지에 여러 구성요소를 효과적이고 보기좋게 배치하는 것을 의미함

레이아웃 구성방법은 1.테이블을 이용 2.div 태그 + css 이용 3.html 레이아웃 태그 + css 이용

시맨틱 웹 관련 요소

개요

시맨틱 태그란 브라우저, 검색엔진, 개발자 모두에게 콘텐츠의 의미를 명확히 설명하는 역할 담당하는 요소

시맨틱 태그를 이용하면 컴퓨터가 HTML 요소의 의미를 보다 명확히 해석하고 데이터로 활용할 수 있는 시맨틱 웹을 실현할 수 있음

시맨틱 태그

header : 웹사이트 로고, 검색폼을 나타내는 영역

nav : 웹사이트를 구성하는 메뉴gnb를 나타내는 영역

main : 웹 사이트의 중요 컨텐츠를 표시하는 영역

section : 중요 컨텐츠를 구성하는 하위 컨텐츠를 그룹화할때 사용

article : 중요 컨텐츠들을 구성하는 하위 컨텐츠가 보여지는 영역

aside : 웹사이트 중요 컨텐츠와 관련없는 내용을 표시하는 영역

footer : 웹사이트 저작권이나 관리자 정보등을 나타내는 영역
