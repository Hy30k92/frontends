개요

모든 웹브라우저는 디폴트 스타일을 가지고 있음 따라서, CSS가 없어도 디폴트 스타일에 의해 기본적인 스타일이 적용되어 보여짐 - user-agent stylesheet

하지만, 브라우저에 따라 디폴트 스타일이 각각 다르기 때문에 스타일이 제각각으로 보여지는 경우가 많음

Hello, World!!

따라서, 이를 초기화reset시켜 다양한 브라우저에서도 동일한 스타일이 적용되도록 해야 함

작업한 결과물이 어떠한 브라우저에서도 동일한 모양을 보여주도록 하는 것을 cross browsing이라 함

단, W3C에서 공식적으로 권장하는 기법은 아니지만, 실무에서 편의에 의해 임의로 실행하는 방식임

브라우저 기본요소의 디자인을 0으로 만들어 각각의 스타일의 차이를 최소화 함 - 서식format 통일

CSS reset 종류

에릭 마이어eric meyer의 css reset : 최초의 css 리셋, 15년이 지난 지금도 많이 사용. 모든 요소에 대해 초기화 시키는 것이 성능과 출력에 이슈가 존재할 수 있음 (https://meyerweb.com/eric/tools/css/reset)

normalize reset : 모든 요소를 0으로 고정적으로 설정하는 방법에서 탈피한 reset. 브라우저의 디폴트 스타일을 통일시키는 방향으로 css를 reset 함. user-agent stylesheet가 표준이 생긴 탓에 탄생하게 된 reset (https://necolas.github.io/normalize.css)

bootstrap css reset : CSS UI Framework의 하나. 적은 노력으로 화려한 웹 페이지를 만들수 있게 해줌 normalize css에 부트스트랩만의 reset 추가 (https://getbootstrap.com)
